# Задача 1.2 — Витрины данных DM

## Постановка задачи

После заполнения слоя DS исходными данными необходимо рассчитать витрины в слое DM:

- **DM.DM_ACCOUNT_TURNOVER_F** — витрина оборотов по лицевым счетам
- **DM.DM_ACCOUNT_BALANCE_F** — витрина остатков по лицевым счетам

## Витрина оборотов (DM_ACCOUNT_TURNOVER_F)

Содержит обороты по лицевым счетам за каждый день, когда были проводки.

**Процедура:** `ds.fill_account_turnover_f(i_OnDate DATE)`

| Поле | Источник |
|------|----------|
| on_date | Входной параметр i_OnDate |
| account_rk | Счёт из DS.FT_POSTING_F (кредитовый или дебетовый) |
| credit_amount | SUM(credit_amount) из FT_POSTING_F где credit_account_rk = account_rk |
| credit_amount_rub | credit_amount × курс (md_exchange_rate_d.reduced_cource) |
| debet_amount | SUM(debet_amount) из FT_POSTING_F где debet_account_rk = account_rk |
| debet_amount_rub | debet_amount × курс |

Если курса нет — умножаем на 1. Счета без проводок за день в витрину не попадают.

## Витрина остатков (DM_ACCOUNT_BALANCE_F)

Содержит остатки по всем активным счетам за каждый день.

**Стартовые данные:** остатки за 31.12.2017 загружаются из DS.FT_BALANCE_F напрямую.

**Процедура:** `ds.fill_account_balance_f(i_OnDate DATE)`

Алгоритм расчёта зависит от типа счёта (`DS.MD_ACCOUNT_D.char_type`):

| char_type | Формула |
|-----------|---------|
| А (активный) | остаток[вчера] + дебет[сегодня] − кредит[сегодня] |
| П (пассивный) | остаток[вчера] − дебет[сегодня] + кредит[сегодня] |

Все счета, актуальные на дату расчёта, попадают в витрину — даже если оборотов не было.

## Логирование

Обе процедуры пишут в `logs.etl_log`:
- Запись при старте (status = 'started')
- Обновление при успехе (status = 'success', rows_loaded = N)
- Обновление при ошибке (status = 'error', error_message = текст ошибки)

## Расчёт за январь 2018

После создания процедур и загрузки стартовых остатков за 31.12.2017 запускается DAG `bank_dm_january_2018`, который последовательно рассчитывает каждый день с 01.01.2018 по 31.01.2018:

```
turnover[01.01] → balance[01.01] → turnover[02.01] → balance[02.01] → ...
```
