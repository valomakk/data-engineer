# Задание 1.1 — Условие

## Контекст

В некотором банке внедрили новую frontend-систему для работы с клиентами, а также обновили саму базу данных. Большую часть данных успешно перенесли из старых БД в одну новую централизованную БД. Но в момент переключения со старой системы на новую возникли непредвиденные проблемы в ETL-процессе — небольшой период (конец 2017, начало 2018 года) так и остался в старой базе. Старую базу отключили, а не выгруженные данные сохранили в CSV-файлы.

Недавно банку потребовалось построить отчёт по 101 форме. Данные из CSV-файлов тоже нужны. Загрузить их в новую БД не получится из-за архитектурных и управленческих сложностей — нужно рассчитать витрину отдельно. Но для этого сначала нужно загрузить исходные данные из CSV-файлов в детальный слой (DS) хранилища в СУБД PostgreSQL.

---

## Задача

Разработать ETL-процесс для загрузки «банковских» данных из CSV-файлов в соответствующие таблицы СУБД PostgreSQL. Покрыть данный процесс логированием этапов работы и всевозможной дополнительной статистикой. Обратить внимание, что в разных файлах может быть разный формат даты — это необходимо учитывать при загрузке.

---

## Описание таблиц

**DS.MD_ACCOUNT_D, DS.MD_CURRENCY_D, DS.MD_EXCHANGE_RATE_D** — информация о счетах, валютах и курсах валют. Содержат поля `data_actual_date` и `data_actual_end_date` для определения актуальных записей на нужную дату. Идентификаторы записей имеют окончание `_rk` (например, `account_rk`).

**DS.MD_LEDGER_ACCOUNT_S** — справочник балансовых счетов, регулируемый Центральным банком. Определяет к какой главе и разделам относятся счета первого (первые 3 цифры) и второго (первые 5 цифр) порядка.

**DS.FT_POSTING_F** — таблица проводок (операций) в рабочем дне (`oper_date`). Состоит из двух частей: счёт дебета и счёт кредита. Изменяет баланс на сумму проводки.

## Требования к реализации

1. Создать пользователя / схему **DS** в БД
2. Создать в DS-схеме таблицы под загрузку данных из CSV-файлов
3. Логировать начало и окончание работы процесса загрузки в специальную таблицу в схеме **LOGS**
4. После логирования старта — добавить паузу **5 секунд**
5. Создать схему **LOGS** с таблицей для хранения логов
6. Использовать стратегию **Upsert** (вставить или обновить) с первичными ключами:

| Таблица | Первичный ключ |
|---|---|
| DS.FT_BALANCE_F | ON_DATE, ACCOUNT_RK |
| DS.FT_POSTING_F | нет (TRUNCATE перед загрузкой) |
| DS.MD_ACCOUNT_D | DATA_ACTUAL_DATE, ACCOUNT_RK |
| DS.MD_CURRENCY_D | CURRENCY_RK, DATA_ACTUAL_DATE |
| DS.MD_EXCHANGE_RATE_D | DATA_ACTUAL_DATE, CURRENCY_RK |
| DS.MD_LEDGER_ACCOUNT_S | LEDGER_ACCOUNT, START_DATE |
