# Пояснение: Задача 1.3 — Алгоритм 101 формы

## Что такое 101 форма

Банковская отчётность: остатки и обороты по балансовым счетам за месяц.
Регулятор (ЦБ РФ) требует эту форму ежемесячно.

## Группировка по балансовому счёту 2-го порядка

Лицевой счёт (20 символов) → первые 5 символов = балансовый счёт 2-го порядка.

Пример: лицевой счёт `40702810000000001234` → балансовый счёт `40702`.

Все лицевые счета с одинаковыми первыми 5 символами суммируются в одну строку отчёта.

## Рублёвые vs валютные счета

Рублёвые: `currency_code IN ('810', '643')` — оба кода означают рубль (810 — старый ОКБО, 643 — ISO).
Валютные: все остальные.

Поля `_RUB` / `_VAL` / `_TOTAL` — это одни и те же суммы в рублях (balance_out_rub, debet_amount_rub, credit_amount_rub), просто разбитые по типу валюты счёта.

## Входящий и исходящий остаток

- Входящий (`BALANCE_IN`) — остаток на 31.12.2017 (день перед началом периода)
- Исходящий (`BALANCE_OUT`) — остаток на 31.01.2018 (последний день периода)

Берём из `dm.dm_account_balance_f`, которая рассчитана в задаче 1.2.

## Логирование

Используем ту же таблицу `logs.etl_log`. Процедура пишет:
- Старт расчёта (status = 'started')
- Финиш (status = 'success', rows_loaded = количество строк в отчёте)
- При ошибке (status = 'error', error_message = текст)
